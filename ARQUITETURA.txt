SIGNIFICADO CADA ARQUIVO DO BACKEND


SRC: ONDE VIVE 100% DA APLICAÇÃO

main.ts ponto de entrada da  aplicação

app.module.ts é o modulo raiz da aplicação responsável por conectar cada modulo do sistema


AUTH: RESPONSAVEL PELA AUTENTICAÇÃO, LOGIN E TOKENS DE SEGURANÇA

auth.modules.ts: define o modulo de autenticação.

auth.controler.ts: define rotas de autenticação

auth.service.ts: define a logica de autenticação

jwt.strategy.ts: define como o jwt sera validado


USERS:RESPONSAVEL POR USUARIOS, PERFIS E ENTIDADES
user.module.ts: define o modulo de usuaros

user.service.ts: camada de regras de negócios dos usuários

user.enity.ts: define a tabela de usuários no banco


DASHBOARD:RESPONSAVEL POR DADOS AGREGADOS E INDICADORES

dashboard.controller.ts: controlado do painel principal


DATABASE: PASTA DO BANCO DE DADOS

database.module.ts: arquivo responsável pela conexão com banco de dados

tennant-connection.services.ts: responsável por resolver o banco correto por tenant


(RAIZ)
.env: variável de ambiente

package.json: manifesto do projeto
